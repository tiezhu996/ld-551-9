## 项目主要需求

### 项目概述

EduFlow 是一个在线课程学习平台，支持课程发布、学员注册、在线学习、进度跟踪、支付购买等完整流程。平台有三种角色：学员（Student）、讲师（Instructor）、管理员（Admin），各角色拥有不同权限和数据访问范围。

系统采用前后端分离架构：前端 Vue 3 + TypeScript + Element Plus，后端 FastAPI + SQLAlchemy，数据库 PostgreSQL，使用 Redis 做缓存和会话管理，JWT 进行身份认证。

### 核心实体（≥ 4个）

#### 1. 课程 Course

```python
# 后端模型
class Course:
    id: int
    title: str                    # 课程标题
    description: str              # 课程描述（富文本）
    instructor_id: int            # 讲师ID（外键 → User）
    category: str                 # 分类（如"编程"、"设计"、"商业"）
    level: CourseLevel            # 难度：BEGINNER / INTERMEDIATE / ADVANCED
    price: Decimal                # 价格
    cover_image: str              # 封面图URL
    total_lessons: int            # 总课时数（自动计算）
    total_duration: int           # 总时长（分钟，自动计算）
    status: CourseStatus          # 状态：DRAFT / PUBLISHED / ARCHIVED
    rating: float                 # 平均评分（0-5）
    student_count: int            # 学员数（自动统计）
    created_at: datetime
    updated_at: datetime
```

- **业务动作**：
  - 讲师创建课程（状态为 DRAFT）
  - 编辑课程信息
  - 上架课程（DRAFT → PUBLISHED，需管理员审核通过后自动上架）
  - 下架课程（PUBLISHED → ARCHIVED，讲师或管理员操作）
  - 管理员审核课程

#### 2. 章节 Chapter + 课时 Lesson

```python
class Chapter:
    id: int
    course_id: int          # 所属课程（外键 → Course）
    title: str              # 章节标题
    sort_order: int         # 排序序号

class Lesson:
    id: int
    chapter_id: int         # 所属章节（外键 → Chapter）
    title: str              # 课时标题
    type: LessonType        # 类型：VIDEO / TEXT / QUIZ
    content: str            # 内容（视频URL / 文本内容 / 题目JSON）
    duration: int           # 时长（分钟）
    is_free: bool           # 是否免费试看
    sort_order: int
```

- **关系**：一个课程包含多个章节（一对多），一个章节包含多个课时（一对多）
- **业务动作**：
  - 创建章节和课时
  - 拖拽排序（修改 sort_order）
  - 编辑课时内容
  - 删除章节/课时（需检查是否有学员进度关联）

#### 3. 学员注册 Enrollment + 学习进度 Progress

```python
class Enrollment:
    id: int
    user_id: int            # 学员ID（外键 → User）
    course_id: int          # 课程ID（外键 → Course）
    enrolled_at: datetime   # 注册时间
    progress: float         # 总进度百分比（0-100，自动计算）
    last_access_at: datetime # 最后访问时间

class LessonProgress:
    id: int
    enrollment_id: int      # 注册ID（外键 → Enrollment）
    lesson_id: int          # 课时ID（外键 → Lesson）
    completed_at: datetime  # 完成时间
    score: int              # 测验分数（仅 QUIZ 类型）
```

- **关系**：学员选课后创建 Enrollment，每个课时的学习进度记录在 LessonProgress
- **业务动作**：
  - 学员注册课程（免费课程直接注册，付费课程需先完成支付）
  - 完成课时（标记 LessonProgress.completed_at）
  - 查看学习进度（Enrollment.progress 自动根据已完成课时比例计算）
  - 测验评分（QUIZ 类型课时完成后记录 score）

#### 4. 订单 Order

```python
class Order:
    id: int
    order_no: str           # 订单编号（唯一）
    user_id: int            # 用户ID（外键 → User）
    course_id: int          # 课程ID（外键 → Course）
    amount: Decimal         # 订单金额
    payment_method: str     # 支付方式（alipay/wechat/mock）
    status: OrderStatus     # 状态：PENDING / PAID / REFUNDED / CANCELLED
    paid_at: datetime       # 支付时间
    created_at: datetime
    updated_at: datetime
```

- **业务动作**：
  - 创建订单（选课时创建，状态 PENDING）
  - 支付订单（PENDING → PAID，支付成功后自动创建 Enrollment）
  - 退款（PAID → REFUNDED，退款后自动撤销 Enrollment）
  - 取消订单（PENDING → CANCELLED，超时未支付自动取消）

### 横切关注点

#### 1. 认证授权 + RBAC

三种角色：**Student**（学员）、**Instructor**（讲师）、**Admin**（管理员）

- **后端**：
  - JWT 认证中间件：所有 `/api/` 接口需验证 token（除公开接口如课程列表）
  - RBAC 权限装饰器：`@require_role(UserRole.INSTRUCTOR)` 等
  - 数据范围过滤：讲师只能查看/编辑自己的课程，学员只能查看自己的注册和进度
- **前端**：
  - 路由守卫：未登录跳转登录页，角色不匹配跳转 403 页
  - 按钮级权限：根据角色显示/隐藏操作按钮（如讲师才能看到"创建课程"）
  - 请求拦截器：统一在请求头添加 Authorization: Bearer token
- **触达范围**：DB Users 表 → 后端 auth 中间件 → 后端 RBAC 装饰器 → 前端路由守卫 → 前端按钮权限指令 → 前端 store 中的角色状态

#### 2. 支付安全

- **订单状态机**：`PENDING → PAID → REFUNDED`（退款分支）、`PENDING → CANCELLED`（取消分支），状态流转必须经过后端 service 层校验
- **幂等性检查**：支付回调接口必须幂等（相同 order_no 重复回调不重复处理），通过数据库唯一约束 + 状态检查实现
- **后端支付 service**：`PaymentService.process_payment(order_id, payment_info)` — 校验订单状态 → 调用支付（Mock）→ 更新订单状态 → 创建 Enrollment → 返回结果
- **前端支付流程**：点击支付 → 创建订单 → 跳转支付页面（Mock） → 回调确认 → 查询订单状态 → 展示结果
- **异常处理**：支付超时（30 分钟自动取消）、支付失败重试、重复支付防护
- **触达范围**：DB Orders 表 → 后端 PaymentService → 后端异常处理中间件 → 前端支付组件 → 前端请求拦截器统一错误提示

### 全局异常处理

#### 后端统一异常中间件

`backend/app/middleware/error_handler.py`：
- 捕获所有未处理异常，返回统一 JSON 格式：`{ "code": 500, "message": "...", "detail": "..." }`
- 业务异常（如 `CourseNotFoundException`、`PaymentFailedException`）定义在 `backend/app/exceptions/` 目录下
- HTTP 异常统一格式化（400/401/403/404/500）
- 记录异常日志（使用 Python logging 模块）

#### 前端请求拦截器统一错误提示

`frontend/src/utils/request.ts`：
- Axios 响应拦截器：根据 HTTP 状态码统一处理
  - 401：清除 token，跳转登录页
  - 403：提示"无权限访问"
  - 404：提示"资源不存在"
  - 500：提示"服务器错误，请稍后重试"
- 使用 Element Plus 的 `ElMessage` 组件展示错误信息
- 网络超时/断网：提示"网络连接异常"

### 严禁合并职责到单一文件

以下规则必须严格遵守：

1. **model 与 schema 分离**：SQLAlchemy 模型（`models/`）负责数据库映射，Pydantic Schema（`schemas/`）负责请求/响应校验，两者不得合并到同一文件
2. **service 与 controller 分离**：API 路由（`api/v1/`）只负责接收请求和返回响应，业务逻辑在 `services/` 中实现，不得在路由文件中写业务逻辑
3. **store 与类型分离**：前端 store（`stores/`）引用独立的类型定义（`types/`），不得在 store 中内联类型
4. **枚举集中管理**：后端 `constants/enums.py` 和前端 `constants/enums.ts` 各维护一份枚举，两份内容必须保持一致，其他文件 import 使用
5. **中间件独立**：`error_handler.py` 和 `audit_log.py` 各自独立，不得合并
6. **组件与页面分离**：共用组件（`components/`）与页面组件（`pages/`）严格分离，共用组件必须被 ≥ 2 个页面使用
7. **异常类独立**：每种业务异常一个文件（`exceptions/`），不得全部写在一个文件中


### 部署要求

项目必须支持 Docker Compose 一键部署，且在 **任意目录名（包括中文目录名）** 下都能正常启动。

docker-compose.yml 要求：
- **不要写 `version:` 字段**
- 顶层声明 `name: ldeduflow`
- 所有服务使用 `container_name` 命名容器，带 `${COMPOSE_PROJECT_NAME:-ldeduflow}` 前缀
- 数据库数据使用命名卷持久化
- 前端通过 Nginx 反向代理后端 API（调用 `/api`，由 Nginx 转发到 `backend:38501`）
- 数据库配置 `healthcheck`，后端 `depends_on` 数据库健康
- 后端配置 `healthcheck`，前端 `depends_on` 后端健康
- 所有可变配置通过环境变量注入

需要提供：docker-compose.yml, .env, .env.example, frontend/Dockerfile, backend/Dockerfile, frontend/nginx.conf

### 代码存放规则

这是前后端分离项目。
前端代码放在 `frontend/`
后端代码放在 `backend/`
数据库脚本放在 `database/`
